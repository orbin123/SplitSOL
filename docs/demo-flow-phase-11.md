# SplitSOL Demo Flow

Use this script for the Phase 11 demo video. Keep the pace brisk and avoid pauses while wallets or confirmations are in progress.

## Setup

- Start from a fresh install with no local app data.
- Make sure Phantom is installed on the Android device.
- Have a second demo identity ready for the friend perspective.
- Use a wallet state that can complete the AutoPay flow on demand.

## Walkthrough

1. Open the app from a fresh install.
   Say: "This is a clean install of SplitSOL."

2. Tap `Connect Phantom Wallet`.
   Say: "SplitSOL uses Phantom for wallet-based identity and settlement."

3. Approve the wallet request in Phantom.
   Say: "After approval, onboarding continues automatically."

4. Enter a display name and continue.
   Say: "First-time users only need a wallet and a display name."

5. Land on Home and pause on the empty state.
   Say: "A new account starts with zero credit, zero debt, and no groups yet."

6. Create a group named `Dinner` with an emoji.
   Say: "I'll create a dinner group for tonight's split."

7. Add a friend via QR scan.
   Say: "Contacts can be added instantly by scanning a SplitSOL QR code."

8. Add a split:
   Say: "Now I'll add Pizza for 20 USDC, paid by me, split equally."

9. Show the balances tab in the group.
   Say: "SplitSOL immediately calculates that my friend owes 10 USDC."

10. Open the settle flow or narrate the friend perspective.
    Say: "From the debtor's side, the pending debt appears in the settle screen."

11. Trigger AutoPay.
    Say: "AutoPay can swap available SOL into USDC and settle without manual token juggling."

12. Wait for the success screen.
    Say: "Once the transaction confirms, SplitSOL shows the result and links directly to Solana Explorer."

13. Tap the explorer link briefly, then return.
    Say: "Every settlement can be verified on-chain."

14. Open transaction history.
    Say: "The confirmed payment is now stored in local transaction history."

15. Open notifications.
    Say: "The counterparty also receives a notification for the settlement flow."

16. Return to Home.
    Say: "Home updates automatically, so balances always reflect the latest state."

## Must-show checkpoints

- Fresh install to connected wallet flow
- Empty home state with 0 owed and 0 owe
- Group creation for `Dinner`
- QR-based friend add
- Pizza split for `20.00 USDC`
- Balances showing `10.00 USDC`
- AutoPay settlement
- Success screen with explorer link
- Transaction history entry
- Notification received
- Updated home balance
